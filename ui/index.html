Overview

PerfLens is a remote Linux performance profiler with a real-time web UI. Drop the agent on any Linux device (ARM or x86), point it at a PID, and watch flame graphs, function tables, perf stat metrics, and line-level annotated source update live in your browser.

No frontend frameworks. No pip dependencies. No Docker. Pure Python stdlib on the server, plain HTML/CSS/JS for the UI.

Quick Start

1. Start the Server

On the machine where you want to view profiles:

python3 server/perflens_server.py \
    --source-dir /path/to/source \
    --binary     /path/to/myprogram \
    --port       9999 \
    --http-port  8080

Or use a pre-built release tarball:

tar xf perflens-server-*-linux-x86_64.tar.gz
./perflens-server --source-dir /path/to/src --binary /path/to/binary

Then open http://localhost:8080 in your browser.

2. Start the Agent

On the target Linux device you want to profile. Two connection patterns:

# Option A: Agent connects to server (--server mode)
python3 perflens_agent.py --server <server-ip>
./perflens-agent --server <server-ip>          # C agent

# Option B: Agent listens, server connects via UI wizard (--listen mode)
python3 perflens_agent.py --listen
./perflens-agent --listen                       # C agent

With --server, the agent connects automatically and the UI switches when data arrives. With --listen, use the Live Debug wizard to connect to the agent's IP:port.

For headless/scripted use (no UI, no server):

./perflens-agent --output result.txt --pid <PID>
./perflens-agent --output - --pid <PID>        # stdout

Server Options

OptionDefaultDescription
--port PORT9999TCP port the agent connects to
--http-port PORT8080HTTP port for the web UI
--source-dir DIR.Root of the source tree for line annotation
--binary PATH—Unstripped binary (enables addr2line source mapping)
--map PATH—GNU ld linker map file (optional symbol fallback)
--path-map FROM=TO—Rewrite compile-time paths to local paths
--addr2line PATH—Custom addr2line binary
--max-samples N500000Ring buffer cap before oldest samples drop
--inline / --no-inlineonEnable/disable inline function resolution
--import FILE—Import a perf.data file at startup as a session

Agent Modes

The agent has three run modes (pick one):

ModeDescription
--listenDaemon: bind port, wait for the server to connect in. Use the UI wizard to connect.
--server HOSTDaemon: connect out to server. Reconnects automatically with exponential backoff.
--output FILEHeadless: collect once, write to file. Use - for stdout. Requires --pid.

Agent Options

OptionDefaultDescription
--pid PID—PID of the process to profile (required for --output; set via UI wizard in daemon modes)
--port PORT9999TCP port (listen or connect)
--frequency HZ99perf record sampling frequency
--duration SECS8Length of each collection round
--rounds N1Number of rounds (--output mode only, Python agent)

UI Walkthrough

Landing Page

Choose Live Debug to connect to an agent via the setup wizard, or Saved Sessions to browse previous profiling sessions. If an agent is already running with --server, the UI switches automatically when it connects.

Setup Wizard

  1. Connect — Enter the agent's IP and port (default 9999)
  2. Perf — Auto-probes supported perf events and call-graph modes
  3. Process — Pick a PID from the running process list or enter manually
  4. Binary — (Optional) Path to unstripped binary and source directory for line-level annotation
  5. Options — Sampling frequency (Hz) and collection duration (seconds)
  6. Start — Review settings and begin profiling

Profiling View

  • Functions tab — Ranked table of hottest functions by self% and total%. Click a row to view source.
  • Source tab — Line-level annotated source with heat-colored sample counts.
  • Flame Graph tab — Interactive SVG flame graph. Click to zoom, double-click to reset. Use the search bar to highlight functions.
  • Sessions tab — List of saved sessions. Replay any session or import a perf.data file.

Device Health Strip

When the agent sends system metrics, a collapsible strip shows live CPU, memory, temperature, load, and per-process stats with sparkline charts.

Troubleshooting

perf_event_paranoid too high

The agent warns at startup if /proc/sys/kernel/perf_event_paranoid > 1. Fix with:

sudo sysctl -w kernel.perf_event_paranoid=1

No function names

Compile with -g and do not strip. file ./myprogram should say not stripped and with debug_info.

No source line mapping

Check that --binary points at the exact unstripped binary and --source-dir contains the source files. Use --path-map /build/src=/home/me/src when build paths differ.

Agent can't connect

The server must be reachable on --port. Test with: nc -zv <server-ip> 9999

Container: perf record is empty

Some container environments strip perf capabilities. A system-wide perf record -a usually works as a fallback.

Slow startup (6–12 seconds)

Call-graph probing tests fp, dwarf, then lbr in sequence. This is normal on first connection.

Export Options

Use the Export button in the header to download:

  • Flamegraph SVG — The current flame graph as a standalone SVG file
  • Collapsed Stacks — Brendan Gregg format, compatible with FlameGraph tools
  • Session JSON — Full session data including all samples and metadata
